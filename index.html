“If I could do it all again, I'd be a plumber.”

-- Albert Einstein

Define the entire data context for your project in a single yaml file. Wherever the source. Whatever the format.

Namely:

  1. define data processing pipes:
    1. where is your data coming from (internet, file, db, memory, etc.)
    2. how should your data be pre-processed
  2. define taps by
    1. giving short id's to your datasets
    2. use them to get your data into R using a single, consistent API
